🔗 Test Liens Projets

Vérification du fonctionnement des liens dans les projets

Bouton "Accéder" → Ouvre le lien du projet depuis le Panel Admin

🎯 Objectif

Problème: Le bouton "Accéder" dans l'Espace Participant ne ouvrait pas les liens des projets

Solution: Modifier le bouton pour ouvrir le lien (URL) saisi dans le Panel Admin

Comportement: Si projet a un lien → ouvre le lien | Si pas de lien → affiche détails

🔧 Modifications Apportées

1. Ajout Champ projectUrl au Backend

ParticipantProject.js Schema

// Project URL/Link
projectUrl: {
  type: String,
  trim: true
},

2. Modification Transformation API

❌ AVANT

projectUrl: apiProject.files?.[0]?.url

✅ APRÈS

projectUrl: apiProject.projectUrl || apiProject.files?.[0]?.url

3. Fonction handleAccessProject Améliorée

✅ Nouvelle Logique

const handleAccessProject = (project: Project) => {
  console.log('Accès au projet:', project.title);
  
  // Si le projet a un URL, l'ouvrir dans un nouvel onglet
  if (project.projectUrl && project.projectUrl.trim()) {
    console.log('🔗 Ouverture du lien projet:', project.projectUrl);
    window.open(project.projectUrl, '_blank', 'noopener,noreferrer');
  } else {
    // Sinon, ouvrir les détails du projet
    console.log('📋 Ouverture des détails du projet (pas de lien)');
    setSelectedProject(project);
    setShowUploadModal(true);
  }
};

4. Bouton Dynamique avec Icônes

✅ Bouton Adaptatif

{project.projectUrl && project.projectUrl.trim() ? (
  <>
    <ExternalLink className="w-4 h-4" />
    <span>Ouvrir lien</span>
  </>
) : (
  <>
    <FileText className="w-4 h-4" />
    <span>Détails</span>
  </>
)}
